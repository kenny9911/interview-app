# 30 · Internationalization — End-to-End Multi-Language Architecture

> **Status:** Authoritative design spec (lead architect). Implements the multi-language contract for **viva** spanning `mobile/ ↔ server/ ↔ agent/`.
> **Scope:** UI text + interview (LLM) language + ASR (Deepgram STT) + TTS (Cartesia / ElevenLabs), as one coherent locale model.
> **Authority:** Subordinate to [`15-decisions.md`](./15-decisions.md) (the ADR). This doc extends **D0** (language gating) and **D1** (env-driven model/voice IDs); where it conflicts with the ADR, the ADR wins. Companion to [`30-voice-architecture.md`](./30-voice-architecture.md) and [`40-prompt-system.md`](./40-prompt-system.md).
> **Target languages (priority order):** English (`en`) → Simplified Chinese (`zh-Hans`) → Traditional Chinese (`zh-Hant`) → Japanese (`ja`) → Korean (`ko`). All five are LTR; no RTL work.
> **Single source of truth for cross-package contradictions:** this document. Two upstream slice designs disagreed on the rollout-gate symbol, the migration map, the `verifyEvidence` rule, the gist budget, and whether the agent may fetch config. Those are **resolved here** (§3, §6); implementers follow this doc, not the slice drafts.

---

## 1. Summary & goals

### 1.1 Goals

1. **One canonical BCP-47 locale code travels the whole stack:** `en`, `zh-Hans`, `zh-Hant`, `ja`, `ko`. No package invents its own variant; only the agent maps canonical → vendor codes, at the vendor boundary.
2. **The mobile UI locale *is* the interview language** for MVP — one selector drives both the RN UI and the `language` sent to `POST /v1/configs`.
3. **The brain conducts and outputs in the locale.** Today no prompt instructs any model to respond in the target language (it is only *shown* to the planner as a label). This is the single biggest correctness gap and the core server change.
4. **The agent learns the language from room metadata** — the same channel as `style`/`persona`, constructed before `session.start`. No new round-trip; the agent stays "audio I/O only."
5. **Scoring integrity (`integrity.ts`) works for CJK.** Today it silently strips 100% of CJK evidence and never runs the affect/bias lint on CJK — a hard blocker for the D12 verbatim-evidence contract. Fixed here, gated behind English-green tests.
6. **Every model / voice / language ID is env-driven (D1).** No string literals in code paths; new voices/models are new env vars with defaults.
7. **Rolling rollout gate.** English ships first as a near-no-op refactor; each subsequent language flips a single list entry only once its STT/TTS/voice/prompt/integrity/native-QA slice is green.

### 1.2 Non-goals (explicit)

- **No "awkward-foreigner voice."** It is a non-goal — and an enforced one — to ever speak a language through a voice not natively recorded for it. An English Cartesia voice reading Japanese, or Korean routed through an unverified passthrough, is a **ship blocker**, not a fallback. Enforcement: startup assertions refuse to boot a session whose `(persona × spoken-language)` voice env is unset (§3.6, §8), and no silent fall-through to the English voice UUID for a non-English session.
- **No mid-interview language switching.** The locale is immutable for a session's life. (Leaves the door open for a future decoupled "UI in English, interview in Japanese" mode — additive, not a migration — but out of scope now.)
- **No bundled CJK fonts.** iOS system faces (PingFang/Hiragino/Apple SD Gothic) are used; no multi-tens-of-MB Noto bundle.
- **No Cantonese, Spanish, or any language outside the five targets.** `es` is dropped from the enum.

### 1.3 The full-transcript-capture guarantee

For **every** language, the candidate's *entire* utterance reaches the brain intact — no clipped tails, no first-segment-only commits, no spurious inter-character spaces. This is guaranteed by three independent fixes (full-turn capture, language-aware endpointing, CJK spacing normalization) and proven by `agent/tests/transcript-completeness.test.ts` (§5). It is a hard prerequisite: a CJK report cannot satisfy the D12 verbatim-evidence contract if the transcript it quotes was truncated before it arrived.

## 2. Locale model & codes

### 2.1 The one vocabulary (BCP-47, script-explicit)

| Canonical code | UI language | Spoken language (ASR/TTS) | Written script (LLM output, transcript, report) |
|---|---|---|---|
| `en` | English | English | Latin |
| `zh-Hans` | 简体中文 | Mandarin (`zh`) | Simplified Han |
| `zh-Hant` | 繁體中文 | Mandarin (`zh`) | Traditional Han |
| `ja` | 日本語 | Japanese | Kana/Kanji |
| `ko` | 한국어 | Korean | Hangul |

**Why script subtags (`zh-Hans`/`zh-Hant`) and not region (`zh-CN`/`zh-TW`)?** The distinction we care about app-wide is **script**, not region. A Hong Kong user wants Traditional; a Singapore user wants Simplified. We map to region codes (`zh-CN`/`zh-TW`) only at the Deepgram boundary (§4), the one place region is the vendor's selector.

### 2.2 Simplified vs Traditional — a text/script distinction only

Simplified and Traditional Chinese are the **same spoken Mandarin**. The distinction is purely written.

**Where it MATTERS:**
- UI strings (separate `zh-Hans`/`zh-Hant` catalogs).
- **Deepgram STT code** (`zh-CN` vs `zh-TW`) — Deepgram formats output in the matching script.
- **LLM output script** — the prompt instructs "Simplified" vs "Traditional"; the transcript, reviewer evidence, and report all inherit it.

**Where it COLLAPSES to spoken `zh`:**
- **Cartesia TTS `language`** — one `zh`; the *text* already carries the right script and Cartesia reads either.
- **Persona voice selection** — keyed by `(persona × spoken-language)`; both are Mandarin.
- **Turn-detection / endpointing thresholds** — keyed by spoken language (`zh`).

### 2.3 Two derived keys

Off the canonical code we derive two keys (defined once on the server in `domain.ts`, mirrored independently in the agent's `lang.ts` because packages are independent):

- **`spokenLang(locale)`**: `zh-Hans | zh-Hant → 'zh'`; otherwise identity (`en|ja|ko`). Drives TTS lang, the voice matrix, the EOU/endpointing threshold.
- **`script(locale)`**: `Simplified | Traditional | n/a`. Drives only the LLM output instruction.

### 2.4 UI locale ↔ interview language

**MVP rule: the UI locale *is* the interview language.** Exactly one user-facing selector. The chosen locale simultaneously switches the ~16 RN screens (via i18next `changeLanguage`) and is sent as `language` to `POST /v1/configs`. There is **no separate interview-language state**. The server stores `language` on the config independent of any client UI state, so a future decoupling is additive.

## 3. End-to-end data flow & contract changes

### 3.1 The path

```
[mobile] user picks locale (SetupScreen / ChooseMode pill / DataPrivacy row)
   │  setLocale(locale)              → switches UI (i18next) + persists (AsyncStorage)
   │  api.createConfig({ language }) → canonical BCP-47 string, replaces hardcoded 'en'
   ▼
[server] POST /v1/configs
   │  Zod-validate language ∈ Language enum            (400 invalid_config on unknown)
   │  reject if language ∉ SUPPORTED_LANGUAGES_P0      (409 language_not_supported)
   │  persist InterviewConfig.language
   ▼
[server] POST /v1/sessions
   │  planInterview(config)  ── languageBlock(config.language) in planner prompt ──► plan in target language
   │  ensureInterviewRoom(room, { persona, style, language })   ← metadata gains language
   ▼
[agent] join room → parseLanguage(metadata)   (default 'en' on any miss)
   │  assert (persona × spokenLang) voice env present  → else refuse to start (no English-voice fallback)
   │  construct Deepgram STT (model+lang), makeTts(provider+lang+voice), TurnDetector(explicit threshold)
   ▼
[agent] per turn → POST /v1/sessions/:id/next-turn { candidateText }   (full turn, CJK-normalized)
   ▼
[server] interviewerTurn / reviewAnswer / analyzeInterview
   │  language looked up from config; languageBlock threaded into every prompt builder
   │  integrity.ts guards run script-aware (CJK-safe), affect-lint unions CJK + EN sets
   ▼
[mobile] GET /v1/sessions/:id  ← now returns `language` (caption chip, report font selection)
```

Localized touchpoints: **STT(ja)** + **TurnDetector explicit-ja-threshold** come from `metadata.language` at startup; **`languageBlock(ja)`** makes every line/note/report Japanese; **CJK-safe `verifyEvidence`** keeps ja evidence from being stripped (D12); **`saveStateIfVersion`** preserves D2 unchanged (language never touches `InterviewState`).

### 3.4 Decision — metadata, NOT `TurnReply` (RESOLVES the "agent may fetch config" contradiction)

**The agent learns language from room metadata, full stop. It MUST use metadata and MUST NOT fetch `GET /v1/sessions/:id`.**

- STT and TTS are constructed **before** `session.start`, which is before the first `begin()` call. Language must be known at construction time; a reply-carried language would force an English construct + an audible `updateOptions` re-handshake on turn 1 — at the candidate's first impression.
- Metadata is the channel already carrying `persona`/`style`, set at room creation. Zero new round-trips; the agent stays dumb.
- `GET /v1/sessions/:id.language` exists **solely for mobile** (caption chip, report-font selection). The agent reading it would create a second source of truth that could disagree with metadata. Forbidden.

`TurnReply` stays `{ spokenText, control, ended, index, total }`.

### 3.5 Decision — language stays on `InterviewConfig`, NOT `InterviewState`

`InterviewState` carries the optimistic-concurrency `version` (D2) and is rewritten per turn via `saveStateIfVersion`. Language is **immutable for the session's life**. Denormalizing it onto the hot, version-bumped record adds migration surface and a write-amplification footgun for zero benefit; every server path that needs language already loads config alongside state. The agent never reads `InterviewState`. (A future mid-interview switch would justify a new versioned field with its own ADR — not now.)

### 3.6 Decision — no silent voice fallback (RESOLVES the awkward-foreigner leak path)

When a language is in `SUPPORTED_LANGUAGES_P0`, the agent at startup **asserts that all three persona voice IDs for its `spokenLang` exist** (`VOICE_ARIA_<SPOKEN>`, `VOICE_SAM_<SPOKEN>`, `VOICE_LENA_<SPOKEN>`). If any is missing for a non-English session, the agent **refuses to start that session** rather than falling back to the English `CARTESIA_VOICE_ARIA` UUID. The English fallback is permitted **only** for `en`. Likewise, `assertLiveCreds` requires `ELEVENLABS_API_KEY` whenever `ko` is reachable, failing fast at boot — never at TTS construction after the candidate has joined.

## 4. Voice & ASR selection matrix

### 4.1 Overall strategy

- **ASR: keep Deepgram** (already wired), made language-aware and **pinned per session** (`detectLanguage:false`). Nova-3 has native streaming models for all five targets (verified: `STTLanguages` includes `ja|ko|zh|zh-CN|zh-TW`). **Do not use `language:'multi'`** — Deepgram's code-switch model omits Mandarin and Korean. Seed `keyterm` with the role tech glossary so code-switched English terms inside CJK speech are anchored.
- **TTS: per-language best-of-breed via a `makeTts(lang, persona)` factory**, two providers, both extending the shared `tts.TTS` base so `AgentSession` stays provider-agnostic. **Cartesia `sonic-3`** (already wired, sub-90ms) for `en/zh-Hans/zh-Hant/ja`; **ElevenLabs `eleven_flash_v2_5`** (standalone `@livekit/agents-plugin-elevenlabs`, own `ELEVENLABS_API_KEY`) for `ko`.
- **Korean coverage (the Cartesia gap):** Cartesia's installed plugin types `TTSLanguages` as `en|es|fr|de|pt|zh|ja` — **no `ko`**. Marketing claims sonic-3 speaks Korean, but that would be an untyped, unverified passthrough on a product where the voice *is* the product. Korean therefore uses ElevenLabs with a native-recorded Korean voice. The bundled `inference.TTS` gateway (`elevenlabs/eleven_flash_v2_5`) is the **documented alternative** for LiveKit-Cloud deployments — but **not a silent env fallback** (see §3.6); switching providers is a deliberate, gated config choice via `TTS_PROVIDER_KO`.
- **Turn detection: `inference.TurnDetector` with an EXPLICIT per-language threshold** (see §4.3) — never relying on the STT-reported language round-trip.

### 4.2 Per-language matrix

| Locale | TTS provider | TTS voices (per-persona env → fallback) | TTS lang | STT provider | STT model | STT lang | Endpointing (ms) | Native-accent rationale |

> Voice IDs are dashboard/library UUIDs provisioned per persona per language (15 net-new beyond English). No literals in code (D1) — all via env (§8). Provisioning + native vetting has a named owner and timeline (§9, open decision).

### 4.3 Turn detection — explicit thresholds (RESOLVES the null-language risk)

`inference.TurnDetector` (on-device via `@livekit/local-inference`, already installed; `version: 'v1-mini'`, no LiveKit Cloud) applies per-language EOU thresholds (verified runtime defaults: `en≈0.36, ja≈0.295, ko≈0.4, zh≈0.355`). **It must be given the configured language explicitly** via `ThresholdOverride`/`TurnDetectorOptions` — derived from `spokenLang(metadata.language)` — **not** left to read the STT-reported `language`, which the Deepgram plugin may report as `null` when `detectLanguage:false` is set on a single-language model. Relying on the round-trip would silently apply the English threshold to every CJK session, reintroducing the exact tail-clipping this design fixes, with green (mocked) tests. A test drives the detector with `language:null` from STT and asserts the CJK threshold is still applied (§5).

### 4.4 Endpointing unit bug (real, fixed here)

`agent/src/agent.ts:34-38` defines `ENDPOINTING` in **seconds** (0.7/0.5/0.35) but `EndpointingOptions.minDelay` is **milliseconds** (verified). Today every language — including English — has a ~0ms floor, clipping every tail. The fix converts to ms and makes it language-aware (§4.2 column). **This changes English turn-taking latency from ~0 to a 300ms floor** — a deliberate, called-out behavior change to the shipped English product (Phase 0 is therefore *not* byte-identical on turn timing; see §10). Also add `interruption: { minWords: 2 }` so backchannels (uh-huh / 예 / そうですね) don't truncate.

### 4.5 Agent factory shape (env-driven, D1)

```ts
// agent/src/lang.ts (new) — defaults shown; every value overridable by env.
export type Lang = 'en' | 'zh-Hans' | 'zh-Hant' | 'ja' | 'ko';
export const spoken = (l: Lang) => (l === 'zh-Hans' || l === 'zh-Hant' ? 'zh' : l);

export const STT_MODEL = (l: Lang) =>
  ({ en: process.env.STT_MODEL_EN, 'zh-Hans': process.env.STT_MODEL_ZH, 'zh-Hant': process.env.STT_MODEL_ZH,
     ja: process.env.STT_MODEL_JA, ko: process.env.STT_MODEL_KO }[l]) ?? process.env.DEEPGRAM_MODEL ?? 'nova-3';

export const STT_LANG = (l: Lang) =>
  ({ en: process.env.STT_LANG_EN ?? 'en-US',
     'zh-Hans': process.env.STT_LANG_ZH_HANS ?? 'zh-CN',
     'zh-Hant': process.env.STT_LANG_ZH_HANT ?? 'zh-TW',
     ja: process.env.STT_LANG_JA ?? 'ja', ko: process.env.STT_LANG_KO ?? 'ko' }[l]);

export const TTS_LANG = (l: Lang) => ({ en:'en', zh:'zh', ja:'ja', ko:'ko' }[spoken(l)]);
```

```ts
// agent/src/agent.ts — voice resolution with NO English fallback for non-en
function voiceFor(persona: Persona, l: Lang): string {
  const key = `VOICE_${persona.toUpperCase()}_${spoken(l).toUpperCase()}`; // e.g. VOICE_ARIA_JA
  const v = process.env[key];
  if (!v && l !== 'en') {
    throw new Error(`Missing voice env ${key} for ${l}; refusing to use English voice for a ${l} session`);
  }
  return v ?? process.env.CARTESIA_VOICE_ARIA!; // English-only fallback
}

function makeTts(l: Lang, persona: Persona) {
  if (spoken(l) === 'ko') {
    const provider = process.env.TTS_PROVIDER_KO ?? 'elevenlabs';
    if (provider === 'elevenlabs') {
      return new elevenlabs.TTS({ apiKey: env.ELEVENLABS_API_KEY,
        voiceId: process.env[`ELEVENLABS_VOICE_${persona.toUpperCase()}_KO`]!,
        modelID: env.ELEVENLABS_MODEL ?? 'eleven_flash_v2_5', language: 'ko' });
    }
    return new inference.TTS({ model: process.env.TTS_MODEL_KO ?? 'elevenlabs/eleven_flash_v2_5',
      voice: process.env[`ELEVENLABS_VOICE_${persona.toUpperCase()}_KO`]!, language: 'ko' });
  }
  return new cartesia.TTS({ model: env.TTS_MODEL, language: TTS_LANG(l), voice: voiceFor(persona, l) });
}
```

## 5. Transcript completeness recipe

Three independent root causes, three fixes, one verification test. (All confirmed in code.)

**Root cause A — first-segment commit (the production tail-drop).** The handler forwards on the first `UserInputTranscribed{isFinal:true}` — one STT *segment*, not the turn; a paused answer fires multiple such events. **Fix A:** forward on `ConversationItemAdded` where `item.role==='user'`, reading `item.textContent` (the framework accumulates all final segments and commits once at end-of-turn). Keep `UserInputTranscribed` **only** to publish interim captions (no `nextTurn` call). Guard against re-emitted non-user items and barge-in re-emits.

**Root cause B — premature commit clips CJK clause-pauses.** `turnDetection:'vad'` + the ~0ms endpointing bug ends the turn mid-answer. **Fix B:** `inference.TurnDetector({version:'v1-mini'})` with the **explicit** per-language threshold (§4.3) + correct-unit, language-aware endpointing floors (§4.2). Interview answers are monologues, so `maxDelay` is 3–4s.

**Root cause C — CJK spacing corruption.** The recognizer joins accumulated segments with an ASCII space, injecting spurious spaces between CJK characters across segment boundaries (e.g. `我们用 React 重构` gaining stray spaces). **Fix C (single, specified owner — RESOLVES the cross-package ambiguity):** normalization happens **exactly once, in the agent, inside the turn-forwarding path, immediately before `nextTurn` is called** — collapsing whitespace adjacent to CJK codepoints. The agent then sends the **normalized** string; the server persists *that exact string* and `verifyEvidence` later matches against it. The server does **not** re-normalize the transcript body, so there is one canonical verbatim string end-to-end. (Deepgram `utterance_end_ms=1000` is added as a gap backstop; requires `interimResults`+`vad_events`, both on.)

**Verification — `agent/tests/transcript-completeness.test.ts` (vitest, mocked Deepgram + mocked detector):**
1. **Multi-segment accumulation:** interim → `is_final("我们用")` → `is_final("React 重构了服务")` → `speech_final`, with a 1.5s gap; assert `backend.nextTurn` is called **exactly once** with the full concatenated+normalized string, not the first segment.
2. **Pause does not commit early:** inject a 1.5s mid-utterance silence under the `ko` endpointing profile (min 800); assert no user `ConversationItemAdded` fires until after the final segment + endpointing window.
3. **CJK spacing:** assert no spurious space between adjacent CJK chars across the segment boundary.
4. **Null-language threshold (§4.3):** drive the detector with `language:null` from STT under a `zh` session; assert the CJK threshold is still applied.

### 6.1 `src/domain.ts` — enum, gate, helpers, migration

**Enum + gate (lines 19–21).** Drop `es` and bare `zh`; widen to the five canonical codes. **Keep the symbol name `SUPPORTED_LANGUAGES_P0`** (consumed at `app.ts:89` and `app.ts:134`) and **keep it `['en']`** — widen one entry at a time per §10. (This RESOLVES the rollout-gate contradiction: the gate stays closed; widening the *enum* does **not** open the *gate*. A gate set to all five would accept every CJK config the moment the enum widens, before any vendor/integrity slice is verified — explicitly rejected.)

### 6.3 `src/scoring/integrity.ts` — CJK-correct guards (D12, blocking)

Define **one shared CJK character class** at the top and reuse it everywhere (RESOLVES the copy-paste drift risk across three call sites):

```ts
export const CJK_CHAR = /[぀-ヿ㐀-鿿豈-﫿가-힯]/u;
```

**`normalizeForMatch` (line 6) — keep Unicode letters/numbers; NFKC-fold.** Replaces `[^a-z0-9\s]` (which deleted all CJK). `\p{L}\p{N}` is a strict superset of `a-z0-9`, and NFKC+lowercase are idempotent on ASCII fixtures, so English tests stay green. NFKC is applied identically on both quote and transcript sides (the only place either is normalized for matching).

```ts
export function normalizeForMatch(s: string): string {
  return s.normalize('NFKC').toLowerCase()
    .replace(/[‘’“”'"「」『』]/gu, '')
    .replace(/[^\p{L}\p{N}\s]/gu, ' ')
    .replace(/\s+/gu, ' ').trim();
}
```

**`verifyEvidence` (line 17) — ONE unified substantiveness rule (RESOLVES the two-doc disagreement and the code-switch mis-rejection, P0-5/P0-6).** Count **information units** = CJK codepoints + spaced non-CJK tokens; require ≥3 units. The threshold is unit-based, not a separate `isCjk && <4` branch, so a mostly-English code-switched quote (`React重构`) is judged by total units, not mis-rejected as "too-short CJK."

```ts
function infoUnitCount(normalized: string): number {
  if (!normalized) return 0;
  const cjk = (normalized.match(CJK_CHAR_G) || []).length;
  const spaced = normalized.replace(CJK_CHAR_G, ' ').split(' ').filter(Boolean).length;
  return cjk + spaced;
}
export function verifyEvidence(quote: string, transcript: string): boolean {
  const q = normalizeForMatch(quote);
  if (q === '') return false;
  if (infoUnitCount(q) < 3) return false;               // EN: <3 words; CJK: <3 chars; mixed: combined
  return normalizeForMatch(transcript).includes(q);
}
```

English contract preserved exactly: `'the redesign'` → 2 units → rejected; `'cut drop-off by 18'` → present, ≥3 → accepted. CJK: a 3+ char verbatim span passes; a 1–2 char particle fails. **The unit threshold is validated against the CJK calibration golden set before any CJK language flips** (§6.5, §10) — not shipped as an unverified guess.

**Affect/protected lint (lines 30–49) — hybrid, enforceable, CJK-aware.** English `\b` regexes are no-ops on CJK. Keep them (always-on, for code-switched English affect terms) and **union** a curated, high-precision CJK lexicon. `lintAffect(text, lang)` unions: EN set + the locale's set + (if **any** CJK present) the CJK set — so a wrong/missing `lang` tag still catches CJK affect, and a leaked English term in a CJK report is still caught. The LLM self-critique remains an additional, non-load-bearing layer; the deterministic backstop is what satisfies D12.

```ts
const AFFECT_CJK: RegExp[] = [
  /緊張|紧张|焦慮|焦虑/u, /語氣|语气|声のトーン|목소리\s*톤/u, /口吃|どもり|말더듬/u,
  /眼神接觸|眼神接触|アイコンタクト|아이\s*컨택/u, /肢體語言|肢体语言|ボディランゲージ|보디랭귀지/u,
  /顯得(緊張|不安|沒自信)|显得(紧张|不安|没自信)/u,
];
const PROTECTED_CJK: RegExp[] = [
  /口音|なまり|訛り|억양/u, /年齡|年龄|年齢|나이/u, /外貌|外見|외모/u, /性別|性别|성별/u, /種族|种族|人種|인종/u,
];
export function lintAffect(text: string, lang: string = 'en'): AffectScan {
  const t = text.normalize('NFKC').toLowerCase();
  const hasCjk = CJK_CHAR.test(t);
  const affect = [...AFFECT_EN, ...(AFFECT_BY_LANG[lang] ?? []), ...(hasCjk ? AFFECT_CJK : [])];
  const prot   = [...PROTECTED_EN, ...(PROTECTED_BY_LANG[lang] ?? []), ...(hasCjk ? PROTECTED_CJK : [])];
  const flagged = [...new Set([...affect, ...prot])].filter(re => re.test(t)).map(re => re.source);
  return { clean: flagged.length === 0, flagged };
}
```

> The CJK lexicon is a **translation deliverable per language** (zh/ja/ko), flagged for **native review** as a hard exit-gate artifact. Until a language's lexicon exists and is reviewed, that language stays out of `SUPPORTED_LANGUAGES_P0`.

**`stripAffect` (line 57) — CJK sentence terminators + lang-aware.** Split on Latin `.!?`+space **and** CJK `。！？…．` (no trailing space). Mixed sentences (code-switched) split on whichever terminator appears; fixtures for mixed-script segmentation are required (§6.5).

```ts
export function stripAffect(text: string, lang: string = 'en'): { text: string; changed: boolean } {
  const sentences = text.split(/(?<=[.!?])\s+|(?<=[。！？…．])/u).filter(s => s.length > 0);
  const kept = sentences.filter(s => lintAffect(s, lang).clean);
  return { text: kept.join(' ').replace(/\s+([。！？…．])/gu, '$1').trim(), changed: kept.length !== sentences.length };
}
```

4. **Native voice provisioning & vetting — owner + timeline.** 15 net-new voice IDs (3 personas × zh/ja/ko, plus optional zhTW) each need dashboard provisioning **and** native A/B vetting. This is a schedule blocker disguised as config. **Needs a named owner per language and a date.**
5. **Native register/accent sign-off as a gate.** Korean honorific (합쇼체/해요체) and Japanese keigo (です・ます) are carried by LLM **text**; an LLM can drift into informal register under long context. Each CJK language's exit-gate includes a **native-register acceptance artifact** (the interviewer sustains formal-polite register across a full interview), not just "voice smoke."
6. **Deepgram tier coverage & Claude in-language report quality.** Confirm the Deepgram account has all five languages enabled. Claude's long-structured-JSON report quality for ja/ko is **assumed** equal to English; the language-agnostic-band assertion (§6.5) **measures** it before flip — do not ship on assumption.
7. **Bundled vs system CJK fonts.** Decision: system (no bundle). If brand consistency for CJK display becomes mandatory, bundle one subsetted Noto weight for display only, body stays system.
8. **Self-hosted vs LiveKit Cloud.** `inference.TurnDetector v1-mini` and `@livekit/local-inference` are on-device (no Cloud). The Cloud turn-detector `v1` and the `inference.TTS` gateway require LiveKit Cloud. Confirm `v1-mini` is acceptable (the design assumes it).

## 10. Phased implementation plan

The rollout gate is the single list `SUPPORTED_LANGUAGES_P0` in `domain.ts:21`. A language ships when its entry is added — and only after its STT/TTS/voice/prompt/integrity **and native-QA** slice is green. Each phase is independently shippable and tested with mocked vendors per repo convention.

### Phase 0 — English plumbing (baseline) · `SUPPORTED_LANGUAGES_P0 = ['en']`
Refactor; **no new language enabled**. Behavior identical for `en` **except** turn timing (the endpointing-ms fix, §4.4, is a deliberate called-out change — Phase 0 exit-gate is "all tests green + English voice smoke confirms the 300ms floor feels natural," NOT "byte-identical turn timing").
- **server:** widen `Language` enum to all five (gate stays `['en']`); add `spokenLang`/`scriptInstruction`; add `languageBlock` to all four builders; `interviewerTurnUser`/`cleanFeedback` signature + threading; unified legacy migration; `language` into room metadata; `GET :id` returns `language`; control-token NFKC hardening.
- **server (integrity):** land CJK-safe `normalizeForMatch`/`verifyEvidence`/`lintAffect`/`stripAffect` with the shared `CJK_CHAR`; **English integrity suite must stay green** (run it first).
- **agent:** `parseLanguage` + `lang.ts` env maps + `makeTts` factory + `voiceFor` (English-only fallback) + `inference.TurnDetector` with **explicit threshold** + `ConversationItemAdded` full-turn capture + agent-side CJK spacing normalization + endpointing-ms fix + `interruption.minWords`. For `en`, all codes resolve to today's defaults.
- **mobile:** install libs; `LocaleProvider`; extract `en` catalog; single-option picker driven by `/v1/healthz.languages`; `language: locale` send.
- **tests:** `domain.test.ts`, `integrity-cjk.test.ts` (assert CJK evidence now *verifies* — not yet *supported*), `transcript-completeness.test.ts` (incl. null-language threshold), prompt-threading, control-token round-trip.
- **Independently shippable:** yes (English-only; locale plumbing inert for other codes).

### Phase 1 — Simplified Chinese · add `'zh-Hans'`
- **agent env:** `STT_MODEL_ZH`, `STT_LANG_ZH_HANS=zh-CN`, `TTS_LANG_ZH=zh`, `CARTESIA_VOICE_*_ZH` (asserted); CJK endpointing floors.
- **server:** verify `scriptInstruction('zh-Hans')` yields Simplified; finalize `verifyEvidence` unit threshold against the **zh calibration golden set**; zh CJK affect lexicon — **native review**.
- **mobile:** `zh-Hans` catalogs (native rewrite + human pass); PingFang SC; line-height audit.
- **eval:** `CALIBRATE_CJK` zh golden transcript (evidence-survival + language-agnostic bands).
- **exit gate:** on-device zh-Hans interview; report in Simplified; integrity green for zh; calibration passes; Mainland-native voice + register sign-off.
- **Independently shippable:** yes.

### Phase 2 — Traditional Chinese · add `'zh-Hant'` (lowest incremental cost)
Text-only delta over Phase 1 (same spoken Mandarin pipeline; reuses `zh` voices, **no new TTS work**). `STT_LANG_ZH_HANT=zh-TW`; `zh-Hant` catalogs; PingFang TC; `scriptInstruction('zh-Hant')` Traditional.
- **exit gate:** reviewer/analyst output is **Traditional, not Simplified** (script assertion) + zh-Hant calibration spot-check + **Taiwanese-native accent sign-off** (hard artifact, §9.2). Optional `CARTESIA_VOICE_*_ZHTW` if the default is rejected.
- **Independently shippable:** yes.

### Phase 3 — Japanese · add `'ja'`
- **agent env:** `STT_MODEL_JA`/`STT_LANG_JA=ja`/`TTS_LANG_JA=ja`/`CARTESIA_VOICE_*_JA` (asserted); ja endpointing; explicit ja threshold.
- **server:** ja affect lexicon (native review); `scriptInstruction('ja')` keigo.
- **mobile:** `ja` catalogs (only `_other` plural); Hiragino Sans.
- **eval:** ja golden transcript.
- **exit gate:** full ja interview + report; integrity + calibration green; Japanese-native voice + **keigo register** sign-off.
- **Independently shippable:** yes.

### Phase 4 — Korean · add `'ko'` (second TTS provider)
- **dep:** `npm install @livekit/agents-plugin-elevenlabs` (pinned) — explicit step, gated **before** the enum flip; `assertLiveCreds` requires `ELEVENLABS_API_KEY`.
- **agent env:** `STT_MODEL_KO`/`STT_LANG_KO=ko`; `TTS_PROVIDER_KO=elevenlabs`; `ELEVENLABS_MODEL`; `ELEVENLABS_VOICE_*_KO` (asserted); ko endpointing; explicit ko threshold.
- **server:** ko affect lexicon (native review); `scriptInstruction('ko')` honorific.
- **mobile:** `ko` catalogs; Apple SD Gothic Neo.
- **eval:** ko golden transcript.
- **exit gate:** full ko interview on the second provider; integrity + calibration green; Korean-native voice + **honorific register** sign-off (real keys, device smoke).
- **Independently shippable:** yes (but depends on the Phase-0 `makeTts` factory).
